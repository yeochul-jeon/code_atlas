## 주요 의사결정

### 1. `.mcp.json` 경로: 상대경로 vs 절대경로

**결정**: 상대경로(`./dist/cli/index.js`) 채택

**근거**:
- Claude Code는 `.mcp.json`이 위치한 디렉토리를 기준으로 상대경로를 해석
- 기존 `~/.mcp.json`에는 절대경로(`/Users/cjenm/cjenm/...`)를 사용 → 다른 머신에서 동작 불가
- 상대경로를 사용하면 클론 후 `npm run build`만으로 바로 동작

**불채택 대안**: `npm link` 방식(`"command": "codeatlas"`) — npm link는 명시적 설치 단계가 추가 필요하므로 기각

### 2. `~/.mcp.json` 처리 방법: 내용 유지 vs 파일 삭제

**결정**: 파일 전체 삭제

**근거**:
- 파일 내 `mcpServers`에 `codeatlas` 단 한 항목만 존재 (다른 MCP 없음)
- 빈 JSON 파일(`{ "mcpServers": {} }`)을 남기는 것보다 완전 삭제가 깔끔함

### 3. `.mcp.json` 커밋 여부

**결정**: 커밋 대상 포함 (`.gitignore` 추가 안 함)

**근거**:
- `.mcp.json`은 팀원이 공유해야 하는 개발 환경 설정
- 절대경로를 담고 있다면 gitignore가 적절하지만, 상대경로이므로 커밋해도 머신 독립성 확보됨
- `docs/quickstart.md`에도 커밋 권장으로 명시

## 작업 상세

### A. CodeAtlas MCP 도구로 cjonstyle-nuxt-frontweb 탐색

- **결정**: 결과가 너무 커서 kind별로 분리 조회 (`class`, `method`, `interface`, `field`)
- **시도**:
  1. `find_circular_deps` → 순환 의존성 없음
  2. `find_dead_code` (전체) → 결과 120만 chars 초과, 파일 저장됨
  3. `find_dead_code --kind class` / `--kind method` / `--kind interface` 로 분리 조회
  4. `field`는 여전히 결과 초과 → 별도 파일 저장
- **오류**: `field` 결과가 23만 chars 초과 (MCP 토큰 제한)
- **해결**: 해당 결과는 요약만 제공하고 추후 좁혀서 조회하도록 안내
- **메모**: 미사용 심볼 총계 — class 29개, method 76개, interface 169개

### B. codeatlas 분석 아키텍처 파악

- **결정**: 대화 중 "LSP인가요?" 질문이 나와 소스 직접 탐색
- **시도**: `src/indexer/tree-sitter/parser.ts`, `graph-pipeline.ts`, `indexer.ts` 직접 독해
- **결론**:
  | 레이어 | 기술 | 용도 |
  |--------|------|------|
  | 파싱 | Tree-sitter | AST → 심볼/참조 추출 |
  | 저장 | SQLite | 심볼/ref/파일 관계 |
  | 그래프 | Kuzu | 상속/순환 의존성 분석 |
  | 벡터 | Embedding | semantic_search |
- **핵심**: LSP가 아닌 **정적 구문 분석** → 런타임 다형성은 탐지 불가 (dead_code가 "potentially"인 이유)

## 대화 중 주요 논의

### LSP vs Tree-sitter 분석 전략

사용자가 "LSP 기반인가요?" 질문. LSP는 런타임 타입 정보를 활용하는 반면, codeatlas는
Tree-sitter 기반 순수 구문 분석. 이 차이가 `find_dead_code` 결과에 "potentially"가 붙는
이유를 설명했다 — 동적 require, 런타임 다형성에 의한 참조는 감지 불가.

### `~/.mcp.json` vs 프로젝트 루트 `.mcp.json` — Claude Code 탐색 동작

Claude Code는 `.mcp.json`을 CWD에서 부모 디렉토리까지 거슬러 탐색하는 구조. 홈 디렉토리는
모든 프로젝트의 공통 조상이므로 사실상 전역 설정이 된다. 공식 quickstart 문서가 `~/.mcp.json`을
가이드한 것이 이 문제의 근본 원인 — 문서도 함께 수정했다.
